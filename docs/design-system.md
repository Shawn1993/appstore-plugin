# Design System: 提审交付包 / 审计报告页

> 交付页与审计页模板的设计规范。改 `skills/appstore/assets/*.html` 的样式前先读这份，保持口径一致。

## 1. 主题与氛围
复刻 App Store Connect 后台的"工作台"气质：克制、密度中等（6/10）、几乎无动效。用户对照它往 ASC 里粘贴内容，视觉必须安静，信息必须一眼可扫。

## 2. 色彩与角色
- **Canvas**（#f5f5f7 / 暗 #161617）— 页面底
- **Surface**（#ffffff / 暗 #1d1d1f）— 卡片。禁用纯黑
- **Ink**（#1d1d1f / 暗 #f5f5f7）主文本；**Ink-2**（#6e6e73）次级
- **Hairline**（#e8e8ed）— 结构线，层级靠它而不是阴影
- **唯一强调色**：苹果蓝（#0071e3 / 暗 #2997ff），只用于可交互元素
- 语义色（独立于强调色）：绿 #23a94a 通过/健康、黄 #c98600 等待/风险、红 #e30000 必拒/未解决

## 3. 排版
- 全部系统字体栈（-apple-system + PingFang SC）——这是对 ASC 的刻意复刻，不引外部字体
- 层级靠字重不靠字号：h1 650/21px，卡题 600/17px，正文 400/14px
- 数字一律等宽（字符计数、构建号、日期）：ui-monospace / tabular-nums
- 禁：UI 文本用衬线（App 图标里的品牌字除外）、emoji 当视觉标记

## 4. 组件
- **卡片**：1px hairline 边框 + 10px 圆角，无阴影——层级靠边框与留白
- **字段**：label 上（含 `字数/上限` 等宽小字）+ value 下 + hover 才浮现的复制按钮（按下 1px 下沉）
- **状态**：彩点 + 文字，单一标识（不再叠左色条）；审计条目左侧 3px 语义色条是唯一例外
- **审计条目**：粗体标题（说人话的后果）→ 一两句说明 → 底部一行元信息（依据链接左、处置状态右）
- **已核通过**：两列 ✓ 清单，不用药丸堆

## 5. 布局
- 左侧栏 230px 固定（App 图标 + 分组导航 + 数量角标），内容区 max-width 1200 居中
- <760px collapse：侧栏变横滑一行，内容单列；宽内容（截图行）自身横滚，页面不横滚

## 6. 动效
- 只有 0.2s 透明度渐入 + 按钮按压位移；`prefers-reduced-motion` 全部关闭
- 禁：弹跳、外发光、渐变大字、装饰性常驻动画

## 7. 反模式（禁）
纯黑 #000、卡片投影堆叠、双重状态标识、3 列等宽卡片阵、AI 文案腔（"赋能/无缝/一站式"入 UI 文案）、占位假数据上线
