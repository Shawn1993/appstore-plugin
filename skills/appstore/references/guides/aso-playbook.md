# ASO 与上线后运营手册（全部方法零外部 API 依赖）

## 1. 元数据字段规则（写文案前必读）

**索引权重**：标题(30字符) > 副标题(30) > 关键词字段(100，隐藏)。描述/推广文本/What's New **不参与 iOS 搜索索引**，只影响转化。

**去重铁律**：标题、副标题、关键词字段三者之间**不重复任何词**——Apple 分字段独立索引，重复=白烧字符。

**关键词字段**：逗号分隔**不加空格**；只写**单数**（单复数自动同索引）；不放 app 名（自动加）、品类名、"app"、"free"、竞品名（违规）；按 量×相关性 排序填满 100 字符。中文场景：CJK 无空格分词，按字/顿号切词考虑。

**标题公式**：`[品牌] - [主关键词]` 或 `[主关键词] [品牌]`（品牌无名时关键词打头）；填满 30；要自然可读。

**描述结构**：前 170 字符（折叠前唯一可见）放最强 hook，忌 "Welcome to…" 开头 → 社会证明 → 4-6 条利益 bullet → 三步用法 → CTA。

**推广文本 170 字符改动免审核**——节日/热点/活动即时更新的唯一通道。

## 2. 关键词研究（种子→评估→打分→分桶）

- 扩展源：App Store 搜索自动补全（试 `[词] app` / `best [词]`）、竞品排名词、同义与用户口语（描述"问题"而非"方案"）、错拼缩写
- 机会分 = 量×0.4 + (100−难度)×0.3 + 相关性×0.3
- 四桶：Primary 3-5（进标题/副标题）、Secondary 5-10、Long-tail 10-20（易排名填字段）、Aspirational 3-5（长期追踪）
- 免费数据源：iTunes Search API（`itunes.apple.com/search?term=X&country=cn&entity=software`，结果顺序≈排名代理，可 cron 存库监控）+ ASC「App 分析」真实搜索来源词。付费估算量（Appeeky $8/月起）有真实曝光后再考虑
- 本地化关键词**不是翻译**：每个市场单独重跑研究

## 3. 截图与 icon 转化规则

- 产品页停留 3-6 秒；**前 3 张截图决定 ~80% 转化**（详见 references/guides/screenshots.md）
- Slot1 = Hook（利益标题+核心 UI；禁欢迎页/登录页/设置页），每张一个利益点、标题 4-6 词、字号 ≥60px
- icon：**最多 2 个元素、不放文字**，60×60 小尺寸下自测清晰；深浅双模式都要有对比；提交 1024×1024 PNG 无透明无圆角。好 icon 提点击率 20-40%
- App Preview 视频：15-30 秒、静音自动播、**只能是 app 实录画面**（出现真人出镜/外部 logo/"Download"字样会因预览真实性被拒）

## 4. 评分弹窗策略

- `SKStoreReviewRequest` 每 365 天最多真弹 3 次（Apple 控制）——省着用
- 只在"价值时刻"后弹（完成核心任务/达成目标后），永不在报错/崩溃后弹
- 门槛全满足才弹：会话≥3、装机≥3 天、完成过激活事件、本版本未评过
- **Pre-prompt 过滤**：先自绘"用得开心吗？"→ Yes 才触发原生弹窗，No 导向意见反馈。平均 +0.3~0.8 星

## 5. 评论管理

- 差评回复框架 HEAR：致谢→共情→回应问题→给出路径（版本修复后更新回复）
- 版本更新可在回复里告知"已修复"，是把 1 星翻成 4 星的主要手段

## 6. 编辑精选（中国区值得主动做）

- 自荐入口：developer.apple.com/contact/app-store/promote/
- 编辑权重：设计质量 > Apple 新技术采用（Widget/Live Activity 等）> 独特价值与故事 > 时效 > 评分 4.5+
- Pitch 结构：app+链接 / 一句话卖点 / 新版亮点 / 用了哪些 Apple 技术 / 故事角度（why now）/ 高清素材
- 时机：大更新前 2-4 周、Apple 活动或节令文化时刻；评分 <4.0 或刚发过 buggy 版本别投
- **In-App Events 是自助精选位**：事件名≤30 字符（参与搜索索引！）；素材官方规格——事件卡横版 16:9（最小 1920×1080、最大 3840×2160），详情页竖版 9:16（最小 1080×1920、最大 2160×3840），jpg/png，视频同比例；事件最长 31 天；已装用户在商店内可见事件卡（再互动触达）；每月≥1 个维持算法资格；提前 3-5 天提审。事件增删改可用 `asc app-events list/view/create/update/delete`

## 7. 被拒恢复（补充 preflight 规则库的软技能面）

- 让用户原样贴完整拒信（含 guideline 号），先分类再动手。高频桶：2.1 崩溃/不完整、2.3.x 元数据不符、3.1.1 必须走 IAP、4.2 功能过薄、**4.3 spam（壳/模板类最高危，靠实质差异化解，不靠申诉）**、5.1.x 隐私
- Resolution Center 回复模板：致谢 → 复述被标问题（一句）→ 具体改动列点 → demo 信息（**即使无需登录也主动给操作步骤/视频**，降低审核摩擦）→ 引用新 build 号。规范回复常 24h 内复审
- 铁律：不辩驳 guideline 先承认；别只改元数据重交同一 binary；申诉只用于审核员明显误判，附复现证据
- 加急审核只用于关键 bug/安全修复，营销用途申请会被标记

## 8. A/B 测试（PPO）

- 原生 PPO 只能测 icon/截图/视频（≤3 变体），测不了标题副标题描述；仅自然流量、一次一个测试
- **永远先测第一张截图**（回报最大），其次 icon
- 每变体 ≥1000 曝光才有信号；目标 90-95% 置信；别中途停

## 9. 上线节奏 checklist（浓缩版）

- T-8 周：ASO 地基（关键词/元数据/截图/icon）、press kit
- T-4 周：TestFlight 内测 50+ 人、接崩溃与分析、评分弹窗就位
- T-1 周：提审（留 2-3 天缓冲）、分阶段发布设置
- 上线日：发布→渠道齐发→全天回评论盯崩溃
- 月 1：编辑精选自荐、完整 ASO 审计、开始 A/B 第一张截图
- 健康线：crash-free sessions >99.5%（崩溃率高会丧失编辑精选资格，>2% 是社区经验红线）

## 10. 数字速查

| 事实 | 数值 |
|---|---|
| 元数据改动生效（索引） | 1-3 天 |
| 季节性关键词 | 提前 2 周上、峰后 3-5 天回退 |
| 自定义产品页 | 最多 35 个，审核 1-3 天，评估需每页 ≥1000 浏览 |
| 分阶段发布 | 每阶段观察 24h，崩溃升 0.2% 即暂停 |
| 订阅宽限期 (Billing Grace Period) | 默认关闭；ASC 手动开启并选最长档，多救非自愿流失单 |
| Google Play 差异 | 全描述参与索引、评分永不重置、feature graphic 1024×500 必填 |
